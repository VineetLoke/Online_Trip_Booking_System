Project Documentation
Project Overview

The Online Booking System is a comprehensive web application that allows users to search for and book flights, trains, and hotels. The system provides a user-friendly interface for searching, booking, and managing travel arrangements.

The project was developed as a college project to demonstrate the application of software engineering principles in building a real-world application.

Project Objectives
  • Develop a user-friendly online booking system for flights, trains, and hotels
  • Implement a secure authentication system for users and administrators
  • Create a responsive and intuitive user interface for all devices
  • Design a robust backend API for handling booking operations
  • Implement a database schema for storing user and booking information
  • Ensure the system is secure, reliable, and scalable
Project Scope

The scope of the project includes:

  • User registration and authentication
  • Search functionality for flights, trains, and hotels
  • Booking and payment processing
  • Booking management (view, cancel)
  • User profile management
  • Admin panel for managing users, flights, trains, and hotels

Technical Documentation
System Architecture

The Online Booking System follows a client-server architecture with a RESTful API. The system is divided into three main components:

  • Frontend: HTML, CSS, JavaScript, and Bootstrap
  • Backend: Node.js with Express.js
  • Database: MongoDB
Architecture Diagram
Component Interaction

The components interact with each other as follows:

  • The frontend sends HTTP requests to the backend API
  • The backend API processes the requests, interacts with the database, and returns responses
  • The database stores and retrieves data as requested by the backend
Security Architecture

The system implements several security measures:

  • Authentication: JWT-based authentication for users and administrators
  • Authorization: Role-based access control for different user types
  • Data Protection: Password hashing using bcrypt
  • Input Validation: Server-side validation of all inputs
  • HTTPS: Secure communication between client and server
Database Design

The Online Booking System uses MongoDB, a NoSQL document database, to store and manage data. The database schema consists of the following collections:

  • users: Stores user information
  • flights: Stores flight information
  • trains: Stores train information
  • hotels: Stores hotel information
  • bookings: Stores booking information
  • admins: Stores administrator information
Database Schema

For detailed information about the database schema, please refer to the Database section of the Project Tracker.

Data Relationships

The database uses a mix of embedding and referencing to represent relationships between entities:

  • Embedding: Used for data that is always accessed together (e.g., traveler information in bookings)
  • Referencing: Used for relationships between major entities (e.g., user to booking)
Indexing Strategy

Indexes are created on fields that are frequently used in queries to improve performance:

  • Single-Field Indexes: For simple queries on one field
  • Compound Indexes: For queries that filter on multiple fields
  • Unique Indexes: To enforce data integrity

Authentication

The backend implements JWT-based authentication for users and administrators. The authentication flow is as follows:

  1. User submits login credentials (email and password)
  2. Backend verifies credentials against the database
  3. If credentials are valid, backend generates a JWT token
  4. Token is returned to the client and stored in localStorage
  5. Client includes token in Authorization header for subsequent requests
  6. Backend verifies token for protected routes
Error Handling

The backend implements a centralized error handling mechanism to ensure consistent error responses. Errors are categorized as:

  • Validation Errors: Invalid input data
  • Authentication Errors: Invalid or expired tokens
  • Authorization Errors: Insufficient permissions
  • Resource Errors: Resource not found or already exists
  • Server Errors: Internal server errors

Deployment Guide

This section provides instructions for deploying the Online Booking System to a production environment.

Prerequisites
  • Node.js (v14 or higher)
  • MongoDB (v4 or higher)
  • npm or yarn
  • A web server (e.g., Nginx or Apache)
Backend Deployment
  1. Clone the repository: git clone https://github.com/username/online-booking-system.git
  2. Navigate to the backend directory: cd online-booking-system/backend
  3. Install dependencies: npm install
  4. Create a .env file with the following variables:
    • PORT=3000
    • MONGODB_URI=mongodb://localhost:27017/online_booking_system
    • JWT_SECRET=your_jwt_secret_key_here
    • JWT_EXPIRES_IN=24h
    • NODE_ENV=production
  5. Start the server: npm start
Frontend Deployment
  1. Navigate to the frontend directory: cd ../frontend
  2. Configure the API endpoint in js/main.js to point to your backend server
  3. Deploy the frontend files to your web server
Database Setup
  1. Start MongoDB: mongod
  2. Run the database setup script: node database_setup.js
Production Considerations
  • Security: Use HTTPS, secure cookies, and proper authentication
  • Performance: Implement caching, compression, and CDN for static assets
  • Scalability: Consider load balancing and database sharding for high traffic
  • Monitoring: Set up logging and monitoring for the application
  • Backup: Implement regular database backups
User Documentation
User Guide

The User Guide provides instructions for using the Online Booking System. It covers all the features and functionalities available to users.

Getting Started
  1. Registration: Create a new account by clicking on the "Register" button and filling in the required information.
  2. Login: Log in to your account using your email and password.
  3. Home Page: The home page provides quick access to search for flights, trains, and hotels.
Searching for Travel Options
  1. Flights: Navigate to the Flights page, enter source, destination, and date, and click on "Search".
  2. Trains: Navigate to the Trains page, enter source, destination, and date, and click on "Search".
  3. Hotels: Navigate to the Hotels page, enter location, check-in/out dates, and guests, and click on "Search".
Making a Booking
  1. Select Option: From the search results, select the flight, train, or hotel you want to book.
  2. Enter Details: Fill in the traveler information and any additional details required.
  3. Payment: Enter payment information and complete the booking.
  4. Confirmation: Receive a booking confirmation with details of your booking.
Managing Bookings
  1. View Bookings: Navigate to the Profile page and click on "Booking History" to view all your bookings.
  2. Cancel Booking: From the Booking History, select a booking and click on "Cancel Booking" to cancel it.
Administrator Guide

The Administrator Guide provides instructions for administrators to manage the Online Booking System. It covers all the features and functionalities available to administrators.

Getting Started
  1. Login: Log in to the admin panel using your admin credentials.
  2. Dashboard: The dashboard provides an overview of the system, including users, bookings, and inventory.
Managing Users
  1. View Users: Navigate to the Users section to view all registered users.
  2. Edit User: Click on a user to view and edit their details.
  3. Delete User: Click on the "Delete" button to remove a user from the system.
Managing Inventory
  1. Flights: Navigate to the Flights section to view, add, edit, or delete flights.
  2. Trains: Navigate to the Trains section to view, add, edit, or delete trains.
  3. Hotels: Navigate to the Hotels section to view, add, edit, or delete hotels.
Managing Bookings
  1. View Bookings: Navigate to the Bookings section to view all bookings in the system.
  2. Edit Booking: Click on a booking to view and edit its details.
  3. Cancel Booking: Click on the "Cancel" button to cancel a booking.
