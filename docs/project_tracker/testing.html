Testing Overview

The Online Booking System has undergone comprehensive testing to ensure functionality, reliability, and user experience. The testing strategy includes unit testing, integration testing, and user acceptance testing.

Unit Testing

Testing individual components

Integration Testing

Testing component interactions

User Acceptance Testing

Testing user scenarios

Testing Approach
Unit Testing

Unit tests focus on testing individual components in isolation. For the Online Booking System, unit tests were created for:

  • Backend API Endpoints: Testing each API endpoint for correct responses
  • Database Models: Testing model validation and methods
  • Authentication: Testing user authentication and authorization
  • Form Validation: Testing input validation for all forms

Unit tests were implemented using Mocha and Chai for the backend, and Jest for the frontend.

Integration Testing

Integration tests focus on testing the interaction between components. For the Online Booking System, integration tests were created for:

  • API-Database Integration: Testing API endpoints with the database
  • Frontend-Backend Integration: Testing frontend components with the backend API
  • Authentication Flow: Testing the complete authentication process
  • Booking Flow: Testing the complete booking process

Integration tests were implemented using Supertest for API testing and Cypress for end-to-end testing.

User Acceptance Testing

User acceptance tests focus on testing the system from a user's perspective. For the Online Booking System, user acceptance tests were created for:

  • User Registration and Login: Testing the user registration and login process
  • Search Functionality: Testing the search functionality for flights, trains, and hotels
  • Booking Process: Testing the complete booking process
  • Payment Process: Testing the payment process
  • User Profile Management: Testing user profile management
  • Booking Management: Testing booking management (view, cancel)

User acceptance tests were conducted manually using a test script and checklist.

Test Environment

Testing was conducted in the following environments:

  • Development Environment: For unit and integration testing
  • Staging Environment: For user acceptance testing
  • Cross-Browser Testing: Chrome, Firefox, Safari, Edge
  • Cross-Device Testing: Desktop, tablet, mobile

// Test user registration validation
describe('User Registration', () => {
  it('should validate user registration input', () => {
    const userData = {
      name: 'John Doe',
      email: 'john@example.com',
      password: 'Password123',
      phone: '+91 9876543210',
      address: '123 Main St, Mumbai'
    };
    
    const result = validateUserRegistration(userData);
    expect(result.isValid).to.be.true;
  });
  
  it('should reject invalid email', () => {
    const userData = {
      name: 'John Doe',
      email: 'invalid-email',
      password: 'Password123',
      phone: '+91 9876543210',
      address: '123 Main St, Mumbai'
    };
    
    const result = validateUserRegistration(userData);
    expect(result.isValid).to.be.false;
    expect(result.errors).to.have.property('email');
  });
  
  it('should reject weak password', () => {
    const userData = {
      name: 'John Doe',
      email: 'john@example.com',
      password: '123',
      phone: '+91 9876543210',
      address: '123 Main St, Mumbai'
    };
    
    const result = validateUserRegistration(userData);
    expect(result.isValid).to.be.false;
    expect(result.errors).to.have.property('password');
  });
});

Example Integration Test
// Test flight booking flow
describe('Flight Booking Flow', () => {
  let token;
  let userId;
  let flightId;
  
  before(async () => {
    // Login user
    const loginResponse = await request(app)
      .post('/api/auth/login')
      .send({
        email: 'john@example.com',
        password: 'Password123'
      });
    
    token = loginResponse.body.token;
    userId = loginResponse.body.user._id;
    
    // Get a flight
    const flightsResponse = await request(app)
      .get('/api/search/flights?source=Mumbai&destination=Delhi&date=2025-07-10');
    
    flightId = flightsResponse.body[0]._id;
  });
  
  it('should create a booking', async () => {
    const bookingData = {
      type: 'flight',
      itemId: flightId,
      travelerInfo: [
        {
          name: 'John Doe',
          age: 35,
          gender: 'Male',
          idType: 'Passport',
          idNumber: 'A1234567'
        }
      ],
      paymentInfo: {
        method: 'credit_card',
        amount: 5500
      }
    };
    
    const response = await request(app)
      .post('/api/bookings')
      .set('Authorization', `Bearer ${token}`)
      .send(bookingData);
    
    expect(response.status).to.equal(201);
    expect(response.body).to.have.property('_id');
    expect(response.body.userId).to.equal(userId);
    expect(response.body.type).to.equal('flight');
    expect(response.body.itemId).to.equal(flightId);
    expect(response.body.status).to.equal('confirmed');
  });
});
