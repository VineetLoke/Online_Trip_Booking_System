Diagrams Overview

Use Case Diagram
Description

The Use Case Diagram illustrates the interactions between users (actors) and the system. It shows what functionalities are available to different types of users.

Key Components
  • Actors: User, Admin
  • Use Cases: Register, Login, Search Flights/Trains/Hotels, Make Booking, Make Payment, View Bookings, Cancel Booking, etc.
  • Relationships: Include, Extend
Purpose

This diagram helps in understanding the system's scope and the functionalities available to different users. It serves as a high-level view of the system's capabilities.

Entity Relationship (ER) Diagram
Description

The ER Diagram represents the data model of the system, showing entities, their attributes, and relationships between entities.

Key Entities
  • User: Stores user information
  • Flight: Contains flight details
  • Train: Contains train details
  • Hotel: Contains hotel information
  • Booking: Stores booking information
  • Admin: Stores admin information
Purpose

This diagram helps in understanding the data structure of the system and how different entities are related to each other. It serves as a blueprint for the database design.

Class Diagram
Description

The Class Diagram shows the structure of the system in terms of classes, their attributes, methods, and relationships between classes.

Key Classes
  • User: Represents a user of the system
  • Flight: Represents a flight
  • Train: Represents a train
  • Hotel: Represents a hotel
  • Booking: Represents a booking
  • Admin: Represents an admin user
Purpose

This diagram helps in understanding the object-oriented structure of the system and how different classes interact with each other. It serves as a blueprint for the implementation of the system.
