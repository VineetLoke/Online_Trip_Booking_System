Middleware

The backend uses several middleware for various purposes:

  • Authentication Middleware: Verifies JWT tokens and protects routes
  • CORS Middleware: Enables cross-origin requests from the frontend
  • Body Parser: Parses incoming request bodies
  • Error Handling Middleware: Catches and processes errors
Authentication & Security

The backend implements several security measures:

  • JWT Authentication: Secure token-based authentication
  • Password Hashing: Passwords are hashed using bcrypt
  • Role-Based Access Control: Different roles have different access levels
  • Input Validation: All inputs are validated before processing
  • HTTPS Support: Secure communication between client and server
Database Integration

The backend connects to MongoDB using Mongoose ODM:

// server.js
const mongoose = require('mongoose');

mongoose.connect(process.env.MONGODB_URI, {
  useNewUrlParser: true,
  useUnifiedTopology: true,
  useCreateIndex: true,
  useFindAndModify: false
})
  .then(() => console.log('MongoDB connected'))
  .catch(err => console.error('MongoDB connection error:', err));
